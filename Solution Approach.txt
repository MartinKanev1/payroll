This assignment was approached by first analyzing the existing payroll implementation and identifying areas that could benefit from better object-oriented design. The original Employee class was refactored into an abstract base class with specialized subclasses (SalariedEmployee, ContractorEmployee, and HourlyEmployee) to separate responsibilities and remove type-checking logic. A Payable interface was introduced so that each employee type can implement its own payroll calculation logic, improving extensibility and readability. The payroll processing logic was also refactored to separate business calculations from output by introducing structured report objects (PayrollReport and PayrollReportEntry). Validation logic was improved by introducing custom exceptions to handle invalid employee data in a clear and maintainable way. The project structure was reorganized into domain-based packages to improve maintainability and clarity. Finally, unit tests were implemented using JUnit to verify payroll calculations, validation rules, and report generation. This ensures the correctness of the system and makes future changes safer and easier to maintain.
